# DefensePulse

A personalized government contracting opportunity feed for defense contractors. Upload your capability statement PDF, and DefensePulse surfaces the most relevant active solicitations from SAM.gov — ranked by match score, enriched with AI summaries, and augmented with competitive intelligence (incumbents, known teaming pairs, and suggested partners).

## What it does

- **Personalized feed** — uploads a capability statement PDF, extracts your NAICS codes, certifications, and focus areas, and ranks opportunities by semantic similarity to your profile
- **AI summaries** — one-sentence summary per opportunity generated by GPT-4o-mini
- **People Intelligence** — click "Who's involved?" on any card to see past award winners at that agency/NAICS, known teaming pairs from subaward data, and suggested partners that complement your profile
- **Filters** — urgency (deadline proximity), set-aside type (8(a), SDVOSB, HUBZone, Total Small Business), and minimum match percentage

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- An [OpenAI API key](https://platform.openai.com/api-keys)
- A [SAM.gov API key](https://sam.gov/profile/details) (free — register at sam.gov, go to Profile → API Keys)

## Setup

**1. Clone the repo**

```bash
git clone https://github.com/macciampa/defensefeed.git
cd defensefeed
```

**2. Create your `.env` file**

```bash
cp .env.example .env
```

Open `.env` and fill in your keys:

```
OPENAI_API_KEY=sk-...
SAM_API_KEY=your-sam-gov-api-key
DATABASE_URL=postgresql://pryzm:pryzm@localhost:5432/pryzm
```

**3. Start the app**

```bash
docker compose up --build
```

First build takes ~2 minutes. Once you see `Ready in Xs` from the frontend, open [http://localhost:3000](http://localhost:3000).

**4. Load real opportunities**

The app ships with demo data so you can explore it immediately. To load real active solicitations from SAM.gov:

```bash
curl -X POST http://localhost:8000/poll
```

This fetches the latest opportunities and populates the database. Takes 30–60 seconds. Check progress at:

```bash
curl http://localhost:8000/poll/status
```

**5. Upload a capability statement**

Navigate to [http://localhost:3000/upload](http://localhost:3000/upload) and upload a PDF capability statement. The app extracts your NAICS codes, certifications, and focus areas and uses them to rank and personalize the feed.

Don't have a capability statement handy? Use any defense contractor's public capability statement PDF — the feed will rank against whatever profile it extracts.

## Architecture notes

**Opportunity matching** uses cosine similarity between a pgvector embedding of the user's capability statement and embeddings of each opportunity's title + description + NAICS + agency. Opportunities are scored with a composite formula that weights similarity, urgency (days until deadline), and recency.

**People Intelligence** (`GET /intel/{sam_id}`) fires two parallel async calls on first expand:
- USASpending.gov (free, no key) — prime award winners at the same agency + NAICS code, plus subaward data for teaming pairs
- SAM.gov entity search — registered small businesses that complement the user's certifications and NAICS profile

Results are cached in the database for 7 days. If SAM.gov is unavailable or rate-limited, the panel falls back to partner suggestions derived from subaward data.

**SAM.gov polling** runs automatically on startup and every 24 hours via APScheduler. The `/poll` endpoint triggers a manual refresh.

## Project structure

```
backend/
  main.py           — FastAPI app, lifespan, CORS
  poller.py         — SAM.gov poll + APScheduler
  routers/
    feed.py         — GET /feed (scored, paginated)
    intel.py        — GET /intel/{sam_id} (People Intelligence)
    profile.py      — POST /profile/upload
  models.py         — SQLAlchemy ORM + migrations
  schemas.py        — Pydantic request/response models

frontend/
  src/app/
    feed/page.tsx   — Main feed with filters
    upload/page.tsx — Capability statement upload
  src/components/
    OpportunityCard.tsx  — Card with inline intel panel
    IntelPanel.tsx       — Who's involved? expandable section
    Sidebar.tsx          — Filters (urgency, set-aside, match %)
```
